29/*!
30\brief The base class for audio drivers: drivers with audio ports.
31
32A concrete derived class will have to be defined with a real audio driver API,
33either callback based one (like CoreAudio, PortAudio..) ones or blocking ones (like ALSA).
34
35Most of the generic audio handing code is part of this class :
36 - concrete callback basedd derived subclasses typically have to Open/Close the underlying audio API,
37 setup the audio callback and implement the Read/Write methods
38 - concrete blocking based derived subclasses typically have to Open/Close the underlying audio API,
39 implement the Read/Write methods and "wraps" the driver with the JackThreadDriver class.
40*/
