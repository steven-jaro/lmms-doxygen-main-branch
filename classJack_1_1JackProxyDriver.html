Detailed Description

\Brief This class describes the Proxy Backend

It uses plain Jack API to connect to an upstream server. The latter is either running as the same user, or is running in promiscuous mode.

The main use case is the multi-user, multi-session, shared workstation:

  • a classic server with hw driver is launched system-wide at boot time, in promiscuous mode, optionally restricted to the audio group
  • in each user session, a jackdbus server is automatically started with JackProxyDriver as master driver, automatically connected to the system-wide one
  • optionally, each user run PulseAudio with a pulse-jack bridge
