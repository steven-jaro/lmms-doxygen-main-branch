50/**
51 Abstract representation of a plugin
52
53 Such a plugin can be an Instrument, Effect, Tool plugin etc.
54
55 Plugins have descriptors, containing meta info, which is used especially
56 by PluginFactory and friends.
57
58 There are also Plugin keys (class Key, confusingly under
59 SubPluginFeatures), which contain pointers to the plugin descriptor.
60
61 Some plugins have sub plugins, e.g. there is one CALF Plugin and for
62 each CALF effect, there is a CALF sub plugin. For those plugins, there
63 are keys for each sub plugin. These keys also link to the superior
64 Plugin::Descriptor. Additionally, they contain attributes that help the
65 superior Plugin saving them and recognizing them when loading.
66
67 In case of sub plugins, the Descriptor has SubPluginFeatures. Those
68 are a bit like values to the sub plugins' keys (in terms of a key-value-
69 map).
70*/
71class LMMS_EXPORT Plugin : public Model, public JournallingObject
72{
73 Q_OBJECT
74public:
75 enum class Type
76 {
77 Instrument, // instrument being used in channel-track
78 Effect, // effect-plugin for effect-board
79 ImportFilter, // filter for importing a file
80 ExportFilter, // filter for exporting a file
81 Tool, // additional tool (level-meter etc)
82 Library, // simple library holding a code-base for
83 // several other plugins (e.g. VST-support)
84 Other,
85 Undefined = 255
86 } ;

128

134 */
