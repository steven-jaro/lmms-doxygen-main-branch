◆ SetupDriverSync()

void Jack::JackDriver::SetupDriverSync ( int ref,
bool freewheel )
protected

In "async" mode, the server does not synchronize itself on the output drivers, thus it would never "consume" the activations.

The synchronization primitives for drivers are setup in "flush" mode that to not keep unneeded activations. Drivers synchro are setup in "flush" mode if server is "async" and NOT freewheel.
